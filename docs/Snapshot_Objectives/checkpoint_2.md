# Snapshot 3 - Checkpoint 2 Objective

## Previous Snapshot Reflection

Snapshot 2 added the simulated Memory Creation and Memory Templates feature. The project documentation was updated to describe how a user would create memories using structured templates. Jira tasks, possible bugs, and TestRail documentation checks were also created for Snapshot 2.

## New Objective

The objective for Snapshot 3 is to simulate the Contacts and Search/Filter features.

## Planned Work

- Define contact management requirements
- Define search and filter requirements
- Add contact and search requirements to the SRS
- Add contact and search design details to the SDD
- Update the Design Spec with Contacts, Contact Details, Search, and Filter sections
- Update the User Manual with contact and search instructions
- Create Jira tasks for the feature
- Create possible bugs and risks
- Create TestRail test cases and a Snapshot 3 report

## Feature Summary

The simulated Contacts feature allows users to organize important people connected to saved memories. The simulated Search and Filter feature allows users to find saved memories by keyword, category, contact, date, or template type.
